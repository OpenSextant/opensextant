Introduction

OpenSextant is a text processing suite geared towards geo-referencing (a.k.a geocoding) documents, spreadsheets and other data. In otherwords, whereever geospatial references are made in some text, the references can be plotted to a map.  OpenSextant extracts and geocodes place name mentions in text. It also supports a basic place name search against standard name catalogs and your own custom name catalogs.  The activity of associating names and codes with geodetic coordinates is also known as gazetteering and indirect referencing.  Direct referencing is the use of the coordinates alone.

Overview of Name Catalogs

The place names used for geocoding include two standard catalogs, NGA Geonames for non-US names, and USGS for US domestic names.  These catalogs together provide world-wide coverage for 260+ countries and territories. The OpenSextant catalog of gazetteers can be extended through its programming interface (TBD)

A name catalog is a mapping of some text, name or code to a geo coordinate.   Such a mapping is also known as a gazetteer.  Fundamentally, these catalogs could be any such data.

Examples:
  • Gazetteering "boston"  ==> may imply Boston, MA within New England within the United States.
  • Geocoding "boston" ==> may yield the coordinate (42.3N, 71.6W)
  • In both cases the text "boston" alone may yield multiple matches in addition to the examples.

Overview of the Process

Using the OpenSextant tool a collection of documents can be geocoded relatively easily. Each document is pushed through the following steps:
  1. Record. Record basic meta data about the file
  2. Parse. Extract and optionally save the body of the text
  3. Scope. Search for country, regional names, and coordinates as clues to scope gazetteering
  4. Match. For each significant phrase in the document place matches are made within scope.
  5. Filter. Filter any name matches which are unlikely to be places, e.g., "Mr. Springfield"
  6. Geocode. Given names that pass the filters, associate point locations to the matches.
  7. Enrich.  Each document contains related info such as dates, authors, classification, report #s, etc.  Such patterns may be worth carrying through the geocoding process.
  8. Summarize.  Each OpenSextant job yields a Summary page collating names and job metrics
Once a collection is processed it can be formatted for common geospatial information system (GIS) tools, for example:
  • GDB: File Geodatabase for ArcInfo, aka FileGDB
  • SHP: Shapefile for ArcInfo and other tools
  • KML: Keyhole format for Google Earth
  • CSV: Comma separated values is viewable in text editors, MS Excel and other tools.
